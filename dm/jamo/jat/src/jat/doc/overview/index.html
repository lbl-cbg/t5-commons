JAT

JAT is a tool designed for archiving analysis results using our tape archiver JAMO. Using JAT also allows validation of submitted files and folders through a user defined template (in JSON or YAML format).

A template must be created for each type of analysis to be archived. The analysis template describes the entire collection of files and data required for a valid, completed analysis. Use of JAT and analysis templates also makes subsequent querying and retrieval of analysis results and metadata possible and efficient, and using properly designed templates will allow querying across different types of analysis (see query list for some typical queries).

When an analysis task is complete, you can use the jat command line tool to submit the analysis by placing a metadata.json file in the root directory of your analysis folder. The metadata.json file contains the collection of files and data produced by a particular analysis for an analysis type described by the template.

Examples of existing analysis templates can be found here
An outline of the process of using 'jat' is:
  • Describe your analysis by creating an analysis template in YAML.
  • Submit your template to a new jat branch (see bitbucket for an explanation)
  • Work with SDM to get your template reviewed and approved
  • Write a script to collect your analysis files and results and package into a 'metadata.json' file
  • Submit the analysis using 'jat' (example command line)
