Metadata

Metadata is the information that describes the file(s) put into JAMO. These metadata are what give JAMO the ability to find and link files together (data provenance). In general the goal is to provide enough metadata about the file so that questions of how the file was generated can be answered years later (e.g., pipeline/software versions, parameters, input files, etc). There are a few top level and organizational-wide fields that should be populated that will establish basic information about your file. These are:
  • file_name - the name of your file you are adding to the archive
  • file_path - the path where this file will live in the archive relative to your group's root directory
  • file_type - the file type (e.g. fastq, assembly, annotation, etc.)
In addition, adding in one of the main organizational IDs that link the file to either the ITS Sequencing Project or the ITS Analysis Project/Task: sequencing_project_id analysis_task_id Data that is produced externally that is not related to any particular sequencing_project_id or analysis_task_id, JAMO provides an external_data flag that can be set. Pipeline/Project level data is defined by the group that owns the pipeline that is creating the data. These data should include as much information as possible about the creation of the file. Ideally this should be enough information such that another person with some familiarity with the pipeline can reproduce the data given the provided metadata.
