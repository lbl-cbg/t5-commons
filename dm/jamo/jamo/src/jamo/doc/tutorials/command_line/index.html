JAMO on the command line

The command line tool used to interact with jamo on the command line is called jamo. You can gain access to this utility by loading the jamo module:
module load jamo
jamo help
The jamo command line tool operates very similar to the git tool. It is used in the following way:
jamo command [file filter] query_name query_args...
The following commands can be used:
  • info: Prints the file info
  • fetch: Makes a request to restore the files
  • link: Makes a request to restore files and creates a link in the current folder to the file
  • report: Will run the custom user report on the retrieved metadata

Info

This command is used to just get file information back on the files returned by the query passed in. It will print a list of files that matched the query criteria in a tab delimited format in the order of:
  • File path: Where the file should be if it is on disk
  • File status: The current status that the file is in. If this is in the PURGED state than it is currently not on disk.
  • Metadata ID: The id of the metadata record, this can be used to get the full information for that record and make a restore request

Link

This command will make a request to restore the files that have matched the query and create a symbolic link in the current directory. This is not a blocking operation so likely these links will point to files that don't exist yet. You can run jamo info on the same query passed to this command to check up on the status of the restore.
