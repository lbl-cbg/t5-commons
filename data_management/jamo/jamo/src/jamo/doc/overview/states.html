JAMO file states

When you put a file into JAMO it doesn't go into the repository right away, instead it gets put into a workflow and thoughout the workflow the file_status key will change states depending where it is in the queue. The states and descriptions are:
  • COPY_READY: This file is currently in the queue to be copied
  • COPY_IN_PROGRESS: This file is currently being copied to the repository
  • COPY_FAILED: This file failed to be copied, if this occurs JAMO will attempt to copy again at a later time
  • COPY_COMPLETE: This file has been copied to the repository
  • BACKUP_READY: This file is in the queue to be sent off to tape
  • BACKUP_IN_PROGRESS: This file is currently being backed up to tape or is waiting to be validated
  • BACKUP_COMPLETE: This file has been sent off to tape and validated sucessfully
  • PURGED: This file is not on disk and only on tape
  • RESTORE_IN_PROGRESS: This file is currently being restored
  • RESTORED: This file is now on disk after it has been purged
  • TAR_READY: This folder is ready to be tared and is in the queue
  • TAR_IN_PROGRESS: This folder is currently being tared
  • TAR_COMPLETE: This folder has been tared sucessfully
